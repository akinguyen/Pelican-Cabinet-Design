Rule file: Output JSON requirements

Use this when preparing the final answer.

Output format:
- Return JSON only.
- Return only the inner object that matches output-shape.txt.
- Do not wrap the answer in an extra outputShape property.
- Do not include markdown, prose, comments, images, or explanations outside JSON.
- Keep JSON valid, compact, and complete.

Wall output:
- Include walls in the required structure.
- Use the correct wallId and wallLabel from the input.
- Use the correct cabinetPlacementMode from the input.
- Every allowed wall should have placements unless notes explain why it cannot.
- Each wall should include placements, notes, and zoneType.

Placement output:
- Each placement must describe the catalog item and position.
- Use consistent leftInches, bottomInches, widthInches, depthInches, and heightInches.
- Include wallFace on every placement.
- Include builtInFillerWidthInches for blind cabinet placements.
- Use topOption as null unless the placement needs sink, surface-cooktop, or front-control-cooktop.
- Notes should be brief and explain only meaningful constraints or design decisions.

Overall output:
- Include layoutType, wallOrder, notes, walls, dimensionNaming, and zonePlan as required by the output shape.
- dimensionNaming should preserve the meaning of widthInches, depthInches, and heightInches.
- zonePlan should briefly explain why each zone is placed where it is.

Before returning:
1. Confirm every catalogId exists in the input catalog.
2. Confirm every placement fits its selected wall face.
3. Confirm no same-wall overlap exists.
4. Confirm no floor-plan corner overlap exists.
5. Confirm fixed objects are preserved.
6. Confirm fillers and end panels follow their rule file.
7. Confirm the JSON parses successfully.
