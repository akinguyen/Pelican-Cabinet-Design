## 8. Fillers and end panels

Keep kitchen runs closed and intentional. Do not leave random gaps.

Fillers and end panels are different and must not be used together for the same exact condition.

Use fillers only to close a gap:

- cabinet to wall
- cabinet to cabinet
- cabinet to appliance/product
- corner clearance
- scribe/fit adjustment

Use:

- accessory-base-filler for base/floor gaps
- accessory-wall-filler for wall/upper gaps

Use end panels only to finish an exposed side facing open air:

- exposed run end
- exposed refrigerator/pantry side
- visible cabinet/product side
- exposed side caused by height/depth/alignment differences
- mid-run exposed side beside a shorter product such as a hood, microwave, oven, shelf, or appliance

Use:

- accessory-base-end-panel for exposed base-category side faces
- accessory-wall-end-panel for exposed wall-category side faces

Decision rule:

- If there is an actual gap to fill between two objects or between an object and a wall, use a filler.
- If there is no gap and the issue is an exposed side facing open air, use an end panel.
- If a cabinet side is against another cabinet/product/wall and not visible, do not use an end panel there.
- If a cabinet end faces open air, use an end panel instead of a filler.
- Do not place both a filler and an end panel in the same exact side/end condition.
- Do not use end panels at shared inside corners where a filler or corner-clearance condition is required.

## 9. Appliance and cabinet workflow

For range/cooktop zones:

- Keep hood or microwave aligned with the range/cooktop.
- Keep the hood centered on the appliance unless a fixedObject prevents it.
- Use balanced cabinets around important cooking appliances when possible.

For sink/dishwasher zones:

- Keep dishwasher directly beside the sink base when possible.

For refrigerator zones:

- Avoid awkward narrow base cabinets beside the refrigerator.

For upper cabinets:

## 10. Corner validation

At every corner, validate the floor plan, not only the elevation.

Check:

- footprint overlap
- door swing
- drawer travel
- appliance clearance
- whether a normal cabinet at the end of one run would be blocked by a perpendicular run
- whether the selected zone/layout creates a cabinet that cannot actually open
- whether a blind/corner cabinet is actually valid

A corner placement is invalid if a perpendicular cabinet/product/panel blocks the door, drawer, or usable opening of the cabinet on the adjacent wall, even when the footprints do not overlap.

Do not place normal base cabinets, drawer bases, pantry cabinets, refrigerators, appliances, or products tight into a corner if a perpendicular adjacent run blocks their usable face.

When a cabinet run enters an inside corner and a perpendicular cabinet run continues from that corner, use the correctly oriented blind cabinet to make the L-shaped corner condition. Do not use a normal base, drawer, pantry, or wall cabinet at that corner if the perpendicular run would block its opening.

For base-level corner runs, use a base blind cabinet.

For wall/upper-level corner runs, use a wall blind cabinet.

The blind cabinet should be the default corner solution whenever two cabinet runs meet at an inside corner and a cabinet is needed at the corner.

Blind cabinets are the default solution for usable inside cabinet corners where two cabinet runs meet, but only when a valid clean L-shaped corner can be made.

Do not force a blind cabinet if:

- the blind side would be exposed to open air
- the visible/usable side would be blocked
- the perpendicular run would need to overlap the blind cabinet
- the wall is not allowed for cabinet placement
- the corner is better solved with clearance, filler, end panel, or a shifted run

If a normal cabinet/product at a corner is blocked by the perpendicular run, choose one of these solutions:

- use the correctly oriented blind cabinet
- pull the blind cabinet farther from the corner within the allowed pull range
- shift the perpendicular run away from the corner
- leave clearance
- change that wall’s zone
- use a different cabinet type
- avoid using that corner for cabinetry

Do not force a storage zone, pantry zone, refrigerator zone, sink zone, or cooking zone into a wall if it creates blocked cabinet doors/drawers at the corner. It is better to redesign the zone placement than to keep a blocked cabinet.

Some corners should not use a corner cabinet or blind cabinet.

Do not use a blind cabinet if:

- the blind area would be exposed to open air
- the visible/usable part would be blocked even after pulling the blind cabinet within the allowed range
- the adjacent perpendicular run would need to overlap it
- a clean non-overlapping L-shaped corner cannot be made
- the wall has cabinetPlacementMode: "none" and cannot receive cabinets

In those cases, use:

- filler
- end panel
- straight run
- shifted perpendicular run
- open clearance
- different zone placement

At connected corners without blind cabinets, start the perpendicular run far enough from the corner to avoid footprint overlap and door/drawer conflicts with the adjacent run.

## 11. Blind cabinet rules

Blind cabinets are the preferred and expected solution for usable inside cabinet corners.

Use blind cabinets to guarantee a clean L-shaped floor-plan corner when base or wall cabinets continue around a perpendicular corner.

Use:

- base-blind-left-cabinet
- base-blind-right-cabinet
- wall-blind-left-cabinet
- wall-blind-right-cabinet

Use a base blind cabinet for base/floor cabinet corners.

Use a wall blind cabinet for upper/wall cabinet corners.

Only use a blind cabinet when:

- the blind side faces the corner/perpendicular run
- the usable side faces open working space
- the adjacent perpendicular run visually aligns with the blind side
- the visible/usable side stays clear
- there is no footprint overlap

For both base and wall blind cabinets, the adjacent perpendicular run must fully cover the blind range while leaving the visible range fully clear. It is not enough for the visible range to be unblocked; the blind range also must not remain exposed.

For right blind cabinets:

- visible/usable part is on the left
- blind part is on the right

For left blind cabinets:

- visible/usable part is on the right
- blind part is on the left

For base blind cabinets:

- 36-inch base blind = 12-inch door + 3-inch built-in filler = 15 inches visible
- 42-inch base blind = 18-inch door + 3-inch built-in filler = 21 inches visible

For wall blind cabinets:

- 36-inch wall blind = 21-inch door + 3-inch built-in filler = 24 inches visible
- 42-inch wall blind = 27-inch door + 3-inch built-in filler = 30 inches visible

widthInches for a blind cabinet is measured from the adjacent corner wall to the far side of the cabinet.

Allowed pull:

- 36-inch blind may be pulled to about 39 inches
- 42-inch blind may be pulled to about 45 inches

Pull the blind cabinet when needed so that the visible/usable part is not blocked by any part of the adjacent perpendicular cabinet/product/object.

The adjacent perpendicular object must align with the blind range, not with the visible range.

A blind cabinet’s built-in filler is part of the blind cabinet. Do not add a separate filler accessory just to represent it.

Every blind cabinet placement must include:

- builtInFillerThicknessInches

Usually use:

- builtInFillerThicknessInches: 3

Inside a blind cabinet front, treat the layout as:

- blind area + door width + built-in filler

Increasing the built-in filler changes the blind/covered range while keeping the total cabinet width coherent.

## 12. Blind cabinet floor-plan validation

For every blind cabinet, base or wall, split the footprint into:

1. Visible/usable range
2. Blind range

The blind cabinet is valid only when all three conditions are true:

1. The visible/usable range is fully clear and operable.
2. The blind range is fully covered/aligned by the adjacent perpendicular cabinet/product/panel.
3. The adjacent perpendicular object does not overlap, slide under, or occupy any part of the blind cabinet footprint.

The visible/usable range:

- must stay fully clear
- must be operable
- must not be covered by another cabinet/product/panel
- must not be blocked by a perpendicular run
- must not fall under any part of the adjacent perpendicular object footprint

The blind range:

- must be fully covered or visually hidden by the adjacent perpendicular run
- must not remain exposed to open air
- must align with the adjacent perpendicular object so the corner forms a clean L shape
- must not require the perpendicular object to overlap the blind cabinet footprint

The adjacent perpendicular object must line up with the blind range, not the visible range.

If any part of the blind range remains exposed, the blind cabinet is not valid yet. Fix it by choosing one of these options:

- increase builtInFillerThicknessInches by a reasonable amount, usually about 4" to 5" if needed, so the visible range becomes larger and the blind range becomes smaller
- pull the blind cabinet farther from the corner within the allowed pull range
- use a 42" blind instead of a 36" blind if it gives a better visible/blind split
- choose a perpendicular wall/base object with a depth that fully covers the blind range without blocking the visible range
- shift the perpendicular run
- redesign the corner or avoid the blind cabinet if a clean L shape cannot be made

Do not use unrealistic built-in filler values. Keep built-in filler thickness within a reasonable cabinet/filler range.

Apply this rule to both:

- base blind cabinets with base/pantry/appliance objects
- wall blind cabinets with wall-category cabinets/products

For wall blind cabinets especially, check that the adjacent perpendicular wall cabinet depth fully covers the wall blind range. A wall blind cabinet is invalid if its blind range is still partly visible, even if its visible range is not blocked.

When drawing floor plans, clearly mark:

- visible/usable range
- blind range
- whether the blind range is fully covered
- no-overlap L-shaped relationship

## 15. Floor-plan object drawing rules — very important

The floor plan must be a true 2D CAD/top-down plan.

Do not draw the floor plan in perspective, isometric, 3D, angled presentation view, or decorative architectural perspective.

Cabinets/products/accessories must never be drawn as skewed parallelograms.

Even if the wall polygon has mitered ends or angled wall thickness, every cabinet/product/accessory footprint must be drawn as a true rectangle in plan view.

Use a deterministic 2D drawing method, preferably code/CAD-style geometry, not freehand image generation.

For every placement, construct the object footprint using wall-local coordinates:

1. Select the placement baseline:

   - Use interiorSide for wallFace: "interior"
   - Use exteriorSide for wallFace: "exterior"

2. Compute the unit wall direction:

   - wallVector = selectedSide.end - selectedSide.start
   - wallDir = normalize(wallVector)

3. Compute the projection direction:

   - Find the vector from selected side midpoint toward the opposite side midpoint.
   - Cabinets project away from the wall body, so use the opposite of that vector.
   - Normalize it as projectionDir.

4. Determine the wall-span:

   - Normal cabinet/product: span = widthInches
   - Filler/end panel: span = thicknessInches

5. Determine the projection depth:

   - Normal cabinet/product: projection = depthInches
   - Filler/end panel: projection = widthInches

6. Convert inches to drawing units using the selected side length:

   - scale = pixelLength(selectedSide) / selectedSide.lengthInches
   - spanPx = span * scale
   - projectionPx = projection * scale
   - Use the same scale for X and Y. Do not use different horizontal/vertical scaling.

7. Compute the four rectangle corners exactly:

   - p1 = selectedSide.start + wallDir * (leftInches * scale)
   - p2 = p1 + wallDir * spanPx
   - p3 = p2 + projectionDir * projectionPx
   - p4 = p1 + projectionDir * projectionPx

8. Draw the footprint using [p1, p2, p3, p4].

9. Validate the geometry:

   - p1 → p2 must be parallel to p4 → p3.
   - p1 → p4 must be parallel to p2 → p3.
   - p1 → p2 must be perpendicular to p1 → p4.
   - Opposite side lengths must match.
   - If not, the object is wrong and must be redrawn.

10. Do not create object footprints by connecting arbitrary wall polygon corners.

11. Do not stretch the object to match wall miter angles.

12. Do not connect the object far edge to the next wall.

13. Do not apply perspective transforms, shearing, or skewing.

14. Wall bodies may be irregular or mitered polygons, but cabinet/product footprints must remain rectangular.

15. If a wall is diagonal, the object rectangle may rotate with that wall, but it must still be a true rectangle.

Correct visual result:

- Wall 1 horizontal objects should look like rectangles sitting on the correct side of Wall 1.
- Wall 2 vertical objects should look like rectangles projecting perpendicular from Wall 2 depending on the selected face.
- Wall 4 horizontal objects should look like rectangles sitting on the correct side of Wall 4.
- No cabinet should look like a slanted parallelogram unless the wall itself is diagonal; even then it must be a rotated rectangle, not a skewed shape.
