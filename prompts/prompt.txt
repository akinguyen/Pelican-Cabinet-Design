## 1. Design mindset

Design like a real kitchen planner, not only like a rule checker.

Create a modern, balanced, standard kitchen by default unless the user requests another style.

Use these design priorities:

1. Function first
   - Keep the refrigerator, sink/dishwasher, and range reasonably connected.
   - Keep dishwasher directly beside the sink when possible.
   - Keep range and hood/microwave aligned.
   - Avoid blocked doors, blocked drawers, unusable corners, and awkward appliance locations.

2. Clean modern appearance
   - Use simple, intentional cabinet runs.
   - Avoid random gaps.
   - Align upper cabinets with base cabinets, appliances, hoods, sinks, refrigerator/pantry groups, fillers, or end panels.
   - Keep range walls visually balanced when possible.
   - Use matching or visually similar cabinet widths around important features such as ranges and sinks when it improves appearance.

3. Smart storage
   - Use pantry cabinets where they make sense.
   - Use drawers for prep/cooking zones.
   - Use trash pull-out near sink/prep when possible.
   - Avoid forcing narrow awkward cabinets beside refrigerators or in bad corner positions.

4. Creative zone planning
   - Do not force zones to follow wall number order.
   - Choose zones based on wall length, fixed objects, appliance fit, corner feasibility, workflow, visual balance, and storage needs.
   - If a planned zone creates a blocked cabinet or bad corner, redesign the zone. Do not keep a bad layout just because the zone idea was reasonable.
   - If a wall is better as prep/storage instead of pantry/cold storage, use it creatively.
   - If a corner cannot be made correctly, shift the run, change cabinet type, use a blind cabinet, or leave clearance instead of forcing cabinetry.

5. Buildable realism
   - Every run should look like something a cabinet installer could actually build.
   - Use fillers only for gaps/clearance.
   - Use end panels only for exposed visible sides.
   - Use blind cabinets correctly at usable inside corners.
   - Validate the floor plan, not only the elevations.

The style can be changed later, but default to a clean modern kitchen with balanced proportions and practical workflow.

## 3. Draw and validate the wall structure before cabinet placement

Before designing cabinets, first understand the wall floor-plan structure from plannerInput.walls.

For each wall:

1. Draw the wall polygon using wallDots and wallShapeDotPairs.
2. Mark interiorSide and exteriorSide separately.
3. Use interiorSide as the cabinet/elevation baseline for wallFace: "interior".
4. Use exteriorSide as the cabinet/elevation baseline only when wallFace: "exterior" is selected.
5. Determine cabinet projection direction from the wall geometry:
   - Find the wall body direction from the selected side toward the opposite side.
   - Cabinets project away from the wall body.
   - For an interior placement, cabinets project from the interior side into the room, opposite the exterior side.
   - For an exterior placement, cabinets project from the exterior side outward, opposite the interior side.
6. leftInches always starts at the selected side’s start point and increases toward that side’s end point.
7. Do not reverse a wall because it looks easier to draw. Respect the selected side’s start and end points.
8. Do not assume walls connect unless their geometry connects.
9. If two wall endpoints do not connect, keep that opening/gap in the floor plan.
10. If a wall has cabinetPlacementMode: "none", still draw the wall structure, but do not place cabinets on that wall.

The floor plan drawing must show the actual wall geometry from the JSON, not a simplified or guessed room outline.

## 4. Kitchen design process after wall validation

After validating the wall structure, design the kitchen in this order:

1. Identify the kitchen shape
   - Determine the layout type from the actual selected wall geometry and usable cabinet faces.
   - Choose the best matching layoutType, such as single-wall, galley, L-shape, U-shape, G-shape, or connected-wall-return.
   - Do not guess the kitchen shape from wall labels alone. Use wallDots, wall connections, openings/gaps, and cabinetPlacementMode.
   - If the selected walls do not form a standard named shape, use connected-wall-return and explain the wall order in notes.

2. Establish the work triangle
   - Identify the three core zones:
     - storage: refrigerator/pantry/cold-storage area
     - prep/washing: sink, dishwasher, trash, and prep surface
     - cooking: range/cooktop and hood/microwave
   - Place the refrigerator, sink, and range so the walking path between them forms a practical triangle when the layout allows it.
   - Aim for each triangle leg to be about 4 to 9 feet, or 48 to 108 inches.
   - If the room shape, wall length, fixedObjects, or catalog limits prevent an ideal triangle, choose the most efficient workflow and explain the limitation in notes.
   - Do not create a perfect triangle if it causes blocked corners, unusable doors/drawers, bad appliance clearance, or an unrealistic cabinet layout.

3. Place corner cabinets first
   - Once the layout boundaries are understood, solve the inside corners before filling the rest of the run.
   - Use correctly oriented blind base cabinets for usable base/floor cabinet corners.
   - Use correctly oriented blind wall cabinets for usable wall/upper cabinet corners.
   - Pull blind cabinets when needed so the visible/usable range stays clear and the blind range aligns with the adjacent perpendicular object.
   - Verify blind-corner conditions before placing normal cabinets next to the corner:
     - visible range clear
     - blind range fully covered/aligned
     - no footprint overlap
     - adjacent perpendicular object depth/projection matches the blind range width when possible
     - doors and drawers can open

4. Work outward from the corner anchors and main appliances
   - After corner conditions are solved, place the main appliances and zone anchors:
     - refrigerator/cold storage
     - sink and dishwasher
     - range/cooktop and hood/microwave
   - Then fill the remaining spaces between corners and appliances with standard base cabinets, drawer bases, pantry cabinets, wall cabinets, and accessories.
   - Prefer standard catalog widths and visually balanced runs.
   - Keep cabinets directly next to one another unless a filler, clearance, appliance, or panel is required.

5. Add fillers intentionally
   - Use filler strips for real gaps, scribe, corner clearance, or out-of-square wall adjustment.
   - A 1" to 2" filler near walls is often useful for real installation tolerance, but use the actual remaining space when the design requires a different filler width.
   - Do not use fillers as exposed side finishes.
   - Do not use both a filler and an end panel for the same exact condition.

6. Plan end panels before finalizing
   - Add end panels wherever an exposed cabinet/product side faces open air.
   - Use end panels at exposed cabinet run ends.
   - Use refrigerator/pantry side panels to create a finished built-in look, especially when supporting or visually enclosing a deep over-refrigerator cabinet.
   - If a dishwasher sits at the end of a base run with no cabinet beside it, add a base end panel to support/finish the dishwasher side and countertop condition.
   - Add wall/base end panels for mid-run exposed sides created by height, depth, or alignment differences, such as taller cabinets next to shorter hoods, microwaves, ovens, shelves, appliances, or open sections.

7. Final layout refinement
   - Check function, appearance, and buildability together.
   - Redesign zones if a cabinet at a corner cannot open.
   - Shift runs, adjust fillers, use blind cabinets, or change cabinet types when needed.
   - The final design should be practical, modern, balanced, and buildable, not just mathematically filled.

- Do not place both a filler and an end panel in the same exact side/end condition.

## 11. Corner validation

At every corner, validate the floor plan, not only the elevation.

Check:

- footprint overlap
- door swing
- drawer travel
- appliance clearance
- whether a normal cabinet at the end of one run would be blocked by a perpendicular run
- whether the selected zone/layout creates a cabinet that cannot actually open
- whether a blind/corner cabinet is actually valid

A corner placement is invalid if a perpendicular cabinet/product/panel blocks the door, drawer, or usable opening of the cabinet on the adjacent wall, even when the footprints do not overlap.

Do not place normal base cabinets, drawer bases, pantry cabinets, refrigerators, appliances, or products tight into a corner if a perpendicular adjacent run blocks their usable face.

When a cabinet run enters an inside corner and a perpendicular cabinet run continues from that corner, use the correctly oriented blind cabinet to make the L-shaped corner condition. Do not use a normal base, drawer, pantry, or wall cabinet at that corner if the perpendicular run would block its opening.

For base-level corner runs, use a base blind cabinet.

For wall/upper-level corner runs, use a wall blind cabinet.

The blind cabinet should be the default corner solution whenever two cabinet runs meet at an inside corner and a cabinet is needed at the corner.

Blind cabinets are the default solution for usable inside cabinet corners where two cabinet runs meet, but only when a valid clean L-shaped corner can be made.

Do not force a blind cabinet if:

- the blind side would be exposed to open air
- the visible/usable side would be blocked
- the blind range cannot be fully covered
- the perpendicular run would need to overlap the blind cabinet
- the wall is not allowed for cabinet placement
- the corner is better solved with clearance, filler, end panel, or a shifted run

If a normal cabinet/product at a corner is blocked by the perpendicular run, choose one of these solutions:

- use the correctly oriented blind cabinet
- pull the blind cabinet farther from the corner within the allowed pull range
- shift the perpendicular run away from the corner
- leave clearance
- change that wall’s zone
- use a different cabinet type
- avoid using that corner for cabinetry

Do not force a storage zone, pantry zone, refrigerator zone, sink zone, or cooking zone into a wall if it creates blocked cabinet doors/drawers at the corner. Redesign the zone placement instead.

Some corners should not use a corner cabinet or blind cabinet.

Do not use a blind cabinet if:

- the blind area would be exposed to open air
- the visible/usable part would be blocked even after pulling the blind cabinet within the allowed range
- the adjacent perpendicular run would need to overlap it
- a clean non-overlapping L-shaped corner cannot be made
- the wall has cabinetPlacementMode: "none" and cannot receive cabinets

In those cases, use:

- filler
- end panel
- straight run
- shifted perpendicular run
- open clearance
- different zone placement

At connected corners without blind cabinets, start the perpendicular run far enough from the corner to avoid footprint overlap and door/drawer conflicts with the adjacent run.

## 12. Blind cabinet rules

Blind cabinets are the preferred and expected solution for usable inside cabinet corners.

Use blind cabinets to guarantee a clean L-shaped floor-plan corner when base or wall cabinets continue around a perpendicular corner.

Use:

- base-blind-left-cabinet
- base-blind-right-cabinet
- wall-blind-left-cabinet
- wall-blind-right-cabinet

Use a base blind cabinet for base/floor cabinet corners.

Use a wall blind cabinet for upper/wall cabinet corners.

Only use a blind cabinet when:

- the blind side faces the corner/perpendicular run
- the usable side faces open working space
- the adjacent perpendicular run visually aligns with and fully covers the blind side/range
- the visible/usable side stays clear
- there is no footprint overlap

For both base and wall blind cabinets, the adjacent perpendicular run must fully cover the blind range while leaving the visible range fully clear. It is not enough for the visible range to be unblocked; the blind range also must not remain exposed.

For right blind cabinets:

- visible/usable part is on the left
- blind part is on the right

For left blind cabinets:

- visible/usable part is on the right
- blind part is on the left

For base blind cabinets:

- 36-inch base blind = 12-inch door + 3-inch built-in filler = 15 inches visible
- 42-inch base blind = 18-inch door + 3-inch built-in filler = 21 inches visible

For wall blind cabinets:

- 36-inch wall blind = 21-inch door + 3-inch built-in filler = 24 inches visible
- 42-inch wall blind = 27-inch door + 3-inch built-in filler = 30 inches visible

widthInches for a blind cabinet is measured from the adjacent corner wall to the far side of the cabinet.

Allowed pull:

- 36-inch blind may be pulled to about 39 inches
- 42-inch blind may be pulled to about 45 inches

Pull the blind cabinet when needed so that the visible/usable part is not blocked and the blind range is fully covered by the adjacent perpendicular cabinet/product/object.

The adjacent perpendicular object must align with the blind range, not with the visible range.

For a clean L-shaped blind-corner condition, size and position the blind cabinet so the blind range matches the projection depth of the adjacent perpendicular cabinet/product whenever possible. The perpendicular object should sit directly in front of the blind range and align with it, without overlapping the blind cabinet footprint.

General rule:
- adjacent perpendicular object depth/projection should equal the blind range width when possible
- if the adjacent object is deeper than the blind range, pull the blind cabinet farther from the corner within its allowed pull range, increase the built-in filler within a reasonable range, use a different blind size, choose a shallower/deeper perpendicular object, or shift the perpendicular run
- if the adjacent object is shallower than the blind range and leaves the blind range exposed, adjust the blind cabinet/filler, choose a deeper perpendicular object, shift the run, or redesign the corner
- use the same logic for base blind cabinets and wall blind cabinets

Examples only:
- if a perpendicular base object is 24" deep, try to make the base blind range 24" so the object aligns with the blind range
- if a perpendicular wall object is 12" deep, try to make the wall blind range 12" so the object aligns with the blind range

The goal is: visible range clear + blind range fully aligned/covered + no overlap + clean L shape.

A blind cabinet’s built-in filler is part of the blind cabinet. Do not add a separate filler accessory just to represent it.

Every blind cabinet placement must include:

- builtInFillerThicknessInches

Usually use:

- builtInFillerThicknessInches: 3

Inside a blind cabinet front, treat the layout as:

- blind area + door width + built-in filler

Increasing the built-in filler changes the blind/covered range while keeping the total cabinet width coherent.

## 13. Blind cabinet floor-plan validation

For every blind cabinet, base or wall, split the footprint into:

1. Visible/usable range
2. Blind range

The blind cabinet is valid only when all three conditions are true:

1. The visible/usable range is fully clear and operable.
2. The blind range is fully visually covered/aligned by the adjacent perpendicular cabinet/product/panel.
3. The adjacent perpendicular object does not overlap, slide under, or occupy any part of the blind cabinet footprint, including the blind range.

Important meaning of “cover the blind range”:

- Cover/alignment means the adjacent perpendicular run forms a clean L shape beside the blind range.
- It does not mean the perpendicular object may physically overlap the blind cabinet.
- The perpendicular object should sit at the front edge or clearance boundary of the blind range, barely touching or aligning with it without sliding under it.
- The blind cabinet footprint and the perpendicular object footprint must remain separate.
- For the cleanest L shape, the blind range width should match the projection depth of the adjacent perpendicular object whenever possible.

The visible/usable range:

- must stay fully clear
- must be operable
- must not be covered by another cabinet/product/panel
- must not be blocked by a perpendicular run
- must not fall under any part of the adjacent perpendicular object footprint

The blind range:

- must be fully visually covered or hidden by the adjacent perpendicular run
- must not remain exposed to open air
- must align with the adjacent perpendicular object so the corner forms a clean L shape
- should match the projection depth of the adjacent perpendicular object whenever possible
- must not overlap with the adjacent perpendicular object
- must not require the perpendicular object to slide under or occupy any part of the blind cabinet footprint

The adjacent perpendicular object must line up with the blind range, not the visible range, but it must line up by adjacency/alignment, not by overlap.

The perpendicular object should sit directly in front of the blind range, with its depth/projection matching the blind range width when possible, while still remaining outside the blind cabinet footprint.

Generic matching logic:

- Compare adjacent perpendicular object projection depth against the blind range width.
- If projection depth equals blind range width, the L-shaped blind corner is clean.
- If projection depth is larger than the blind range, the object may block the visible range or fail to align cleanly. Fix by pulling the blind cabinet farther within its allowed pull range, increasing built-in filler within a reasonable range, using a different blind width, choosing a shallower perpendicular object, or shifting the perpendicular run.
- If projection depth is smaller than the blind range, part of the blind range may remain exposed. Fix by adjusting blind/filler dimensions, choosing a deeper perpendicular object, shifting the run, or avoiding that blind corner.
- This is a proportional matching rule, not a fixed-size rule. Use the actual dimensions from the selected catalog items.

Examples only:
- a 24" deep perpendicular base object works best with an about 24" base blind range
- a 12" deep perpendicular wall object works best with an about 12" wall blind range

If the adjacent perpendicular object overlaps any part of the blind cabinet footprint, including the blind range, the design is invalid. Push or shift the perpendicular object farther from the corner until it no longer overlaps, while still aligning with the blind range and maintaining the clean L-shaped corner.

If any part of the blind range remains visually exposed or the adjacent perpendicular object's depth does not align cleanly with the blind range, the blind cabinet is not valid yet. Fix it by choosing one of these options:

- increase builtInFillerThicknessInches by a reasonable amount, usually about 4" to 5" if needed, so the visible range becomes larger and the blind range becomes smaller
- pull the blind cabinet farther from the corner within the allowed pull range so the blind range matches the adjacent perpendicular object depth when possible
- use a 42" blind instead of a 36" blind if it gives a better visible/blind split
- choose a perpendicular wall/base object with a depth that fully covers and aligns with the blind range without blocking the visible range and without overlapping the blind cabinet footprint
- shift the perpendicular run so it starts directly in front of the blind range at the front edge or clearance boundary without overlapping it
- redesign the corner or avoid the blind cabinet if a clean L shape cannot be made

Do not use unrealistic built-in filler values. Keep built-in filler thickness within a reasonable cabinet/filler range.

Apply this rule to both:

- base blind cabinets with base/pantry/appliance objects
- wall blind cabinets with wall-category cabinets/products

For wall blind cabinets especially, check that the adjacent perpendicular wall cabinet depth fully covers/aligns with the wall blind range without overlapping the wall blind cabinet footprint. A wall blind cabinet is invalid if its blind range is still partly visible, or if the adjacent wall cabinet overlaps the blind range.

When drawing floor plans, clearly mark:

- visible/usable range
- blind range
- whether the blind range is fully covered/aligned
- no-overlap L-shaped relationship

## 16. Floor-plan object drawing rules

The floor plan must be a true 2D CAD/top-down plan.

Do not draw the floor plan in perspective, isometric, 3D, angled presentation view, or decorative architectural perspective.

Cabinets/products/accessories must never be drawn as skewed parallelograms.

Even if the wall polygon has mitered ends or angled wall thickness, every cabinet/product/accessory footprint must be drawn as a true rectangle in plan view.

Use a deterministic 2D drawing method, preferably code/CAD-style geometry, not freehand image generation.

For every placement, construct the object footprint using wall-local coordinates:

1. Select the placement baseline:

   - Use interiorSide for wallFace: "interior"
   - Use exteriorSide for wallFace: "exterior"

2. Compute the unit wall direction:

   - wallVector = selectedSide.end - selectedSide.start
   - wallDir = normalize(wallVector)

3. Compute the projection direction:

   - Find the vector from selected side midpoint toward the opposite side midpoint.
   - Cabinets project away from the wall body, so use the opposite of that vector.
   - Normalize it as projectionDir.

4. Determine the wall-span:

   - Normal cabinet/product: span = widthInches
   - Filler/end panel: span = thicknessInches

5. Determine the projection depth:

   - Normal cabinet/product: projection = depthInches
   - Filler/end panel: projection = widthInches

6. Convert inches to drawing units using the selected side length:

   - scale = pixelLength(selectedSide) / selectedSide.lengthInches
   - spanPx = span * scale
   - projectionPx = projection * scale
   - Use the same scale for X and Y. Do not use different horizontal/vertical scaling.

7. Compute the four rectangle corners exactly:

   - p1 = selectedSide.start + wallDir * (leftInches * scale)
   - p2 = p1 + wallDir * spanPx
   - p3 = p2 + projectionDir * projectionPx
   - p4 = p1 + projectionDir * projectionPx

8. Draw the footprint using [p1, p2, p3, p4].

9. Validate the geometry:

   - p1 -> p2 must be parallel to p4 -> p3.
   - p1 -> p4 must be parallel to p2 -> p3.
   - p1 -> p2 must be perpendicular to p1 -> p4.
   - Opposite side lengths must match.
   - If not, the object is wrong and must be redrawn.

10. Do not create object footprints by connecting arbitrary wall polygon corners.
11. Do not stretch the object to match wall miter angles.
