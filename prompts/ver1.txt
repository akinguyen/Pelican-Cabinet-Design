## 2. Draw and validate the wall structure before cabinet placement

Before designing cabinets, first draw the wall floor-plan structure from `plannerInput.walls`.

For each wall:

1. Draw the wall polygon using `wallDots` in the given wall shape order.

2. Use `wallShapeDotPairs` to understand how the dots connect into the wall body.

3. Mark `interiorSide` and `exteriorSide` separately.

4. Use `interiorSide` as the cabinet/elevation baseline for `wallFace: "interior"`.

5. Use `exteriorSide` as the cabinet/elevation baseline only when `wallFace: "exterior"` is selected.

6. Determine cabinet projection direction from the wall geometry:
   - Find the wall body direction from the selected side toward the opposite side.
   - Cabinets project away from the wall body.
   - For an interior placement, cabinets project from the interior side into the room, opposite the exterior side.
   - For an exterior placement, cabinets project from the exterior side outward, opposite the interior side.

7. `leftInches` always starts at the selected side’s `start` point and increases toward that side’s `end` point.

8. Do not reverse a wall just because it looks easier to draw. Respect the selected side’s start and end points.

9. Do not assume walls form a rectangle or continuous U shape. Use actual `wallDots`, shared endpoints, and gaps.

10. If two wall endpoints do not connect, keep that opening/gap in the floor plan. Do not close it unless the wall geometry actually connects.

11. If a wall has `cabinetPlacementMode: "none"`, still draw the wall structure, but do not place cabinets on that wall.

12. Only after this wall-structure check is complete, place cabinets and validate the floor plan.

The floor plan drawing must show the actual wall geometry from the JSON, not a simplified or guessed room outline.

## 3. Select usable wall faces

Use `cabinetPlacementMode`:

- `none`: do not place cabinets on this wall
- `interior`: interior face only
- `exterior`: exterior face only
- `both`: either or both faces may be used

Use:
- `interiorSide.lengthInches` as the elevation width for interior placement
- `exteriorSide.lengthInches` as the elevation width for exterior placement

Default wall height is 96 inches.

Include only walls where `cabinetPlacementMode` is not `none`. An allowed wall may still return `placements: []` if it should remain empty.

## 7. Dimension naming rules

For normal cabinets/products:
- `widthInches` = left-to-right wall/elevation span
- `depthInches` = projection away from wall
- `heightInches` = vertical height

For fillers/end panels:
- `thicknessInches` = left-to-right wall/elevation span
- `widthInches` = projection/depth away from wall
- do not use `depthInches`

For overlap checking on the same wall:
- normal cabinet/product: check `leftInches + widthInches`
- filler/end panel: check `leftInches + thicknessInches`

## 8. Fillers and end panels

Keep kitchen runs closed and intentional. Do not leave random gaps.

Use:
- `accessory-base-filler` for base/floor gaps
- `accessory-wall-filler` for wall/upper gaps
- `accessory-base-end-panel` for exposed base-category side faces
- `accessory-wall-end-panel` for exposed wall-category side faces

Fillers close gaps.

End panels finish exposed sides.

Use end panels at:
- exposed run ends
- refrigerator/pantry side finishes
- visible cabinet/product sides
- side faces exposed by height differences
- side faces exposed by depth differences
- mid-run exposed sides beside shorter products such as hoods, microwaves, ovens, shelves, or appliances

Do not use end panels as gap fillers.

Do not use end panels at shared inside corners where a filler or corner-clearance condition is required.

## 9. Appliance and cabinet workflow

For range/cooktop zones:
- Keep hood or microwave aligned with the range/cooktop.
- Keep the hood centered on the appliance unless a fixedObject prevents it.
- Use balanced cabinets around important cooking appliances when possible.

For sink/dishwasher zones:
- Keep dishwasher directly beside the sink base when possible.
- Do not separate sink and dishwasher with unrelated cabinets or fillers.

For refrigerator zones:
- Avoid awkward narrow base cabinets beside the refrigerator.
- Use fillers, panels, pantry cabinets, or over-refrigerator storage when supported by the catalog.
- Avoid placing a refrigerator directly adjacent to a blind cabinet.

For upper cabinets:
- Align wall cabinets visually with base cabinets, appliances, hoods, sinks, refrigerators, pantry groupings, windows, fillers, or end panels.
- Avoid unexplained upper-cabinet gaps.

Use the term `pantry cabinet`, not `tall cabinet`.

Pantry cabinets may be stacked by repeating pantry catalogIds at different `bottomInches`.

## 10. Corner validation

At every corner, validate the floor plan, not only the elevation.

Check:
- footprint overlap
- door swing
- drawer travel
- appliance clearance
- whether a normal cabinet at the end of one run would be blocked by a perpendicular run
- whether a blind/corner cabinet is actually valid

Some corners should not use a corner cabinet or blind cabinet.

Do not use a blind cabinet if:
- the blind area would be exposed to open air
- the visible/usable part would be blocked
- the adjacent perpendicular run would need to overlap it
- a clean L-shaped corner cannot be made

In those cases, use:
- filler
- end panel
- straight run
- shifted perpendicular run
- open clearance

At connected corners without blind cabinets, start the perpendicular run far enough from the corner to avoid footprint overlap and door/drawer conflicts.

## 11. Blind cabinet rules

Blind cabinets are only for clean L-shaped corner conditions.

Use:
- `base-blind-left-cabinet`
- `base-blind-right-cabinet`
- `wall-blind-left-cabinet`
- `wall-blind-right-cabinet`

Only use a blind cabinet when:
- the blind side faces the corner/perpendicular run
- the usable side faces open working space
- the adjacent perpendicular run visually covers the blind side
- the visible/usable side stays clear
- there is no footprint overlap

For right blind cabinets:
- visible/usable part is on the left
- blind part is on the right

For left blind cabinets:
- visible/usable part is on the right
- blind part is on the left

For base blind cabinets:
- 36-inch base blind = 12-inch door + 3-inch built-in filler = 15 inches visible
- 42-inch base blind = 18-inch door + 3-inch built-in filler = 21 inches visible

For wall blind cabinets:
- 36-inch wall blind = 21-inch door + 3-inch built-in filler = 24 inches visible
- 42-inch wall blind = 27-inch door + 3-inch built-in filler = 30 inches visible

`widthInches` for a blind cabinet is measured from the adjacent corner wall to the far side of the cabinet.

Allowed pull:
- 36-inch blind may be pulled to about 39 inches
- 42-inch blind may be pulled to about 45 inches

A blind cabinet’s built-in filler is part of the blind cabinet. Do not add a separate filler accessory just to represent it.

Every blind cabinet placement must include:
- `builtInFillerThicknessInches`

Usually use:
- `builtInFillerThicknessInches: 3`

Inside a blind cabinet front, treat the layout as:
- blind area + door width + built-in filler

Increasing the built-in filler changes the blind/covered range while keeping the total cabinet width coherent.

## 12. Blind cabinet floor-plan validation

For every blind cabinet, split the footprint into:

1. Visible/usable range
2. Blind range

The visible/usable range:
- must stay fully clear
- must be operable
- must not be covered by another cabinet/product/panel
- must not be blocked by a perpendicular run

The blind range:
- should align with the adjacent perpendicular run
- should visually form the L-shaped corner condition

The adjacent perpendicular cabinet/product/panel must not:
- overlap the blind cabinet
- slide under the blind cabinet
- occupy any part of the blind cabinet footprint

The blind cabinet and perpendicular object should form a clean L shape at the corner, touching or aligning at the corner clearance boundary without overlap.

Place the adjacent perpendicular run so it lines up with the blind range, not the visible range.

If the perpendicular object would need to overlap the blind cabinet to cover the blind side:
- do not overlap it
- pull the blind cabinet if allowed
- use a shallower perpendicular object
- shift the perpendicular run
- or avoid the blind/corner cabinet

Apply these checks to:
- base blind cabinets with base/pantry/appliance objects
- wall blind cabinets with wall-category objects/products

When drawing floor plans, clearly mark:
- blind cabinet visible/usable range
- blind cabinet blind range
- no-overlap L-shaped relationship
