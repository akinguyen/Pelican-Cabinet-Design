# 08 — Corner Cabinet and Corner Clearance Rules

Corners must be buildable. Do not create an L-shaped corner by letting two normal cabinets crash into each other.

## Valid corner strategies

Use one of these when catalog definitions and wall permissions allow it:

- blind base cabinet
- blind wall cabinet
- lazy susan/corner cabinet if available
- corner filler/clearance strategy
- stop one run before the corner

## Blind cabinets

When the user asks for blind cabinets, try blind cabinets first. The blind side must be covered by the perpendicular run or intended corner condition. The usable door/drawer side must stay clear.

Use the correct left/right blind cabinet for the wall direction and perpendicular run. Do not expose the blind side to open air unless the catalog definition is explicitly intended for that.

## Cabinet placement sides

Both walls participating in a corner must have valid `cabinetPlacementFaceSides` for the faces receiving cabinets. If one wall face is not allowed, do not force a corner cabinet there.
